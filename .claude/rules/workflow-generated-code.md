# Generated Code Pipeline (ts-rs)

## Workflow

Rust structs generate TypeScript types via `ts-rs` for the frontend to consume. This is a one-way pipeline **sourced on the Rust side**.

```
src-tauri/src/models/*.rs      (#[derive(TS)] #[ts(export)])
         │
         │  pnpm generate:types
         ▼
src/types/bindings/*.ts        (generated, committed)
         │
         ▼
src/types/*.ts                 (hand-written, import from ./bindings)
         │
         ▼
src/lib/*.ts  +  src/components/*
```

## Trigger Conditions

When you need to run `pnpm generate:types`:

- Adding a Rust struct (e.g. a new transfer task field)
- Modifying struct fields (rename, change type, add/remove a field)
- Adding/modifying an enum (e.g. adding a new `ErrorCode` variant)
- Adding the Input/Output struct of an IPC command

## Command

```bash
pnpm generate:types
# Equivalent to: cd src-tauri && cargo test export_bindings -- --nocapture
```

Output directory: `src/types/bindings/` (already in git).

## Rust-Side Declaration

```rust
use serde::{Deserialize, Serialize};
use thiserror::Error;
#[cfg(test)]
use ts_rs::TS;

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
#[cfg_attr(test, derive(TS))]
#[cfg_attr(test, ts(export))]
#[serde(rename_all = "SCREAMING_SNAKE_CASE")]
pub enum ErrorCode { /* ... */ }

#[derive(Debug, Clone, Serialize, Deserialize, Error)]
#[cfg_attr(test, derive(TS))]
#[cfg_attr(test, ts(export))]
pub struct AppError {
    pub code: ErrorCode,
    pub message: String,
    #[serde(skip_serializing_if = "Option::is_none")]
    pub detail: Option<String>,
    #[serde(skip_serializing_if = "Option::is_none")]
    pub retryable: Option<bool>,
}
```

**Hard rules**:

- `ts-rs` is in `[dev-dependencies]` (does not pollute the production build)
- Use `#[cfg_attr(test, derive(TS))]` rather than a bare `#[derive(TS)]`
- `#[cfg_attr(test, ts(export))]` triggers generation
- Align serde rename with the TS side:
  - struct fields: `#[serde(rename_all = "camelCase")]` → `camelCase` on the TS side
  - enums: `#[serde(rename_all = "SCREAMING_SNAKE_CASE")]` → `"AUTH_FAILED"` on the TS side
- Pair `Option<T>` with `#[serde(skip_serializing_if = "Option::is_none")]` to avoid serializing null

## TS-Side Consumption

```typescript
// src/types/error.ts (hand-written, re-exports from bindings + adds business types)
import type { ErrorCode as GeneratedErrorCode } from "./bindings/ErrorCode";
import type { AppError } from "./bindings/AppError";

export { ErrorCode } from "./bindings/ErrorCode";
export type { AppError };

// Default error-code copy (hand-written, since the Rust side has none)
export const ERROR_MESSAGES: Record<GeneratedErrorCode, string> = {
  AUTH_FAILED: "认证失败",
  HOSTKEY_MISMATCH: "主机密钥不匹配",
  // ...
};
```

## Consistency Check (binding-checks.ts)

The project has `src/types/bindings/binding-checks.ts`, which validates at the `tsc --noEmit` stage that hand-written types do not drift from the bindings:

```typescript
// Example (pseudo):
type _Check<T, U> = T extends U ? (U extends T ? true : false) : false;

const _profileCheck: _Check<HandwrittenProfile, GeneratedProfile> = true;
```

If a hand-written type drifts from the bindings, tsc errors out. Fix: run `pnpm generate:types`, then adjust the hand-written type.

## CI Validation

The `bindings` job in `.github/workflows/ci.yml` will:

1. Re-run `cargo test export_bindings`
2. `git diff --exit-code src/types/bindings/` — fails if there is any diff
3. Remind the developer that they "forgot to commit the bindings, or changed a Rust struct but didn't run `pnpm generate:types`"

**Forbidden** to manually edit `src/types/bindings/*.ts` — there is already a comment at the top:

```typescript
// This file was generated by [ts-rs](https://github.com/Aleph-Alpha/ts-rs). Do not edit this file manually.
```

## Commit Flow

Standard flow after changing a Rust struct:

```bash
# 1. Change src-tauri/src/models/*.rs
# 2. Generate bindings
pnpm generate:types

# 3. Adjust the corresponding hand-written types (e.g. add entries to ERROR_MESSAGES)
# 4. Run tests
pnpm test:run && cd src-tauri && cargo test

# 5. Commit together (Rust changes + bindings + hand-written types)
git add src-tauri/src/models/ src/types/bindings/ src/types/
git commit -m "feat(transfer): add ..."
```

**Forbidden** to commit only the Rust changes without updating the bindings — CI will break.

## Prohibited List

- **Forbidden** to hand-write `src/types/bindings/*.ts` — auto-generated
- **Forbidden** to move `ts-rs` into `[dependencies]` — dev only
- **Forbidden** to skip `pnpm generate:types` and edit bindings directly (bypasses binding-checks, causing runtime mismatch)
- **Forbidden** to use a bare `#[derive(TS)]` export — use `#[cfg_attr(test, ...)]` to avoid polluting the production binary
- **Forbidden** to change the bindings file names — the TS import paths are stable; renaming triggers large-scale cascading changes
